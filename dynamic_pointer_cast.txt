一、作用简介

std::dynamic_pointer_cast 是 C++11 引入的智能指针工具函数，
用于在 std::shared_ptr 智能指针之间进行安全的“向下转型（downcast）”。

换句话说，它可以在运行时将：

std::shared_ptr<Base>


安全地转换为：

std::shared_ptr<Derived>

二、使用语法
std::shared_ptr<Derived> p2 = std::dynamic_pointer_cast<Derived>(p1);


其中：

p1：是一个指向基类（Base）的 std::shared_ptr；

Derived：是 Base 的派生类；

返回：std::shared_ptr<Derived>

如果转换成功（对象真实类型是 Derived），则返回一个新的 shared_ptr<Derived>；

否则返回一个空指针（nullptr）。

三、使用前提条件
条件	说明
🔹 类之间存在继承关系	Derived 必须继承自 Base
🔹 Base 类必须至少有一个虚函数	因为 dynamic_pointer_cast 依赖 RTTI（运行时类型识别）机制
🔹 被转换的对象真实类型是 Derived 或其子类	否则返回 nullptr
四、运行机制

std::dynamic_pointer_cast 内部会调用 dynamic_cast 进行类型检测；

如果检测通过，则创建一个新的 shared_ptr<Derived>；

这个新指针与原来的 shared_ptr<Base> 共享相同的控制块（即引用计数一致）；

检测失败时，返回一个空指针（nullptr）。

📘 示意图：

Base* ─────►[Derived object]
↑                 │
│                 ▼
shared_ptr<Base>   shared_ptr<Derived>（共享控制块）

引用计数相同，共享生命周期

五、典型使用场景
✅ 1. 处理多态对象的安全类型识别

当我们通过基类指针接收派生类对象时，不确定其真实类型：

struct Base { virtual ~Base() = default; };
struct Dog : Base { void bark() { std::cout << "Woof!\n"; } };
struct Cat : Base { void meow() { std::cout << "Meow!\n"; } };

void speak(std::shared_ptr<Base> animal) {
    if (auto dog = std::dynamic_pointer_cast<Dog>(animal)) {
        dog->bark();
    }
    else if (auto cat = std::dynamic_pointer_cast<Cat>(animal)) {
        cat->meow();
    }
}

✅ 2. 抽象类场景（常用于框架和插件系统）
struct PluginBase {
    virtual void run() = 0;
    virtual ~PluginBase() = default;
};

struct MyPlugin : PluginBase {
    void run() override { std::cout << "MyPlugin running\n"; }
};

std::shared_ptr<PluginBase> plugin = std::make_shared<MyPlugin>();

auto real = std::dynamic_pointer_cast<MyPlugin>(plugin);
if (real)
real->run();


即使 Base 是抽象类（有纯虚函数），也能正常运行。

✅ 3. 安全替代传统裸指针 dynamic_cast

传统写法（容易忘记释放内存）：

Base * base = new Derived();
Derived* d = dynamic_cast<Derived*>(base);


智能指针写法（自动管理内存）：

auto base = std::make_shared<Derived>();
auto d = std::dynamic_pointer_cast<Derived>(base);

六、与其他类型转换的比较
转换函数	类型检查	转换时机	安全性	常用场景
static_pointer_cast	❌ 无检查	编译期	❌（错误转换 = 未定义行为）	明确知道类型正确时
dynamic_pointer_cast	✅ 运行时检查	运行期	✅（失败返回 nullptr）	类型不确定时
const_pointer_cast	N / A	编译期	改变 const 属性	移除 const 限定
reinterpret_pointer_cast(C++20)	❌ 无检查	编译期	❌ 非常危险	极少使用

💡 提醒：
dynamic_pointer_cast 是最“安全”的，但有轻微运行时性能开销（RTTI 查询）。

七、失败情况（返回 nullptr）
std::shared_ptr<Base> base = std::make_shared<Base>(); // 实际不是 Derived
auto derived = std::dynamic_pointer_cast<Derived>(base);

if (!derived) {
    std::cout << "转换失败\n";
}


输出：

转换失败

八、引用计数关系（非常重要）

所有通过 dynamic_pointer_cast、static_pointer_cast 得到的智能指针
都会共享同一个控制块（即同一个引用计数）。

示例：

std::shared_ptr<Base> base = std::make_shared<Derived>();
auto derived = std::dynamic_pointer_cast<Derived>(base);

std::cout << base.use_count() << " " << derived.use_count() << "\n";


输出（假设没有其他指针）：

2 2


✅ 两个智能指针共享同一控制块。
无论销毁哪一个，只要计数不为 0，内存都不会释放。

九、Base 没有虚函数时的后果
struct Base {};
struct Derived : Base {};

std::shared_ptr<Base> base = std::make_shared<Derived>();
auto d = std::dynamic_pointer_cast<Derived>(base);


🚫 编译错误：

error : cannot dynamic_cast ‘base.get()’(of type ‘Base * ’) to type ‘Derived* ’(source type is not polymorphic)


💡 原因：dynamic_cast 依赖 RTTI（运行时类型信息），而 RTTI 只有在类有 虚函数表（vtable） 时才可用。

十、性能注意事项

dynamic_pointer_cast 内部调用 dynamic_cast，有运行时类型检查，性能比 static_pointer_cast 稍低；

但其安全性更高，在不确定对象类型时强烈推荐使用；

若类型转换失败，返回空指针，不会崩溃。

十一、实战建议与最佳实践
场景	推荐
明确知道类型（性能优先）	std::static_pointer_cast
不确定类型（安全优先）	std::dynamic_pointer_cast
想判断某指针对象是否属于某子类	用 dynamic_pointer_cast 检查是否为 nullptr
多态抽象框架	必须保证基类有虚函数
跨平台兼容性（GCC / Clang / MSVC）	都完全支持，无差异
十二、完整示例
#include <iostream>
#include <memory>

struct Base {
    virtual ~Base() = default;
};

struct Derived : Base {
    void hello() { std::cout << "Hello from Derived\n"; }
};

int main() {
    std::shared_ptr<Base> base = std::make_shared<Derived>();

    auto derived = std::dynamic_pointer_cast<Derived>(base);
    if (derived) {
        derived->hello();
        std::cout << "use_count = " << derived.use_count() << "\n";
    }
    else {
        std::cout << "转换失败\n";
    }
}


输出：

Hello from Derived
use_count = 2

📖 十三、总结（核心要点）
项目	内容
功能	在运行时安全地从 shared_ptr<Base> 转为 shared_ptr<Derived>
前提	有继承关系 + 基类有虚函数
结果	成功：共享引用计数；失败：返回 nullptr
内部实现	基于 dynamic_cast
推荐场景	不确定对象真实类型时
性能	有轻微运行时开销（RTTI）
常见错误	基类无虚函数 → 编译错误
